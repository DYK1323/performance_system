# 관리자 사용자 흐름 (Admin Flow)

## 개요

관리자(기획조정팀)는 크게 두 가지 맥락에서 시스템을 사용합니다.

| 맥락 | 설명 | 빈도 |
|------|------|------|
| **A. 최초 설정** | 새 사업(Program)을 만들고 구조를 설계하는 단계 | 사업당 1회 |
| **B. 운영 관리** | 기존 사업을 운영하며 수시로 수행하는 작업 | 상시 |

두 맥락은 진입점은 같으나 이후 흐름이 다릅니다.

---

## A. 최초 설정 흐름 (신규 Program 생성)

순서가 중요합니다. 하위 단계가 상위 단계에 의존하기 때문에 순서대로 설정해야 합니다.

```
┌─────────────────────────────────────────────────────────────┐
│ [A-1] 로그인 (SSO)                                         │
└──────────────────────┬──────────────────────────────────────┘
                       │
┌──────────────────────▼──────────────────────────────────────┐
│ [A-2] 관리자 대시보드                                       │
│     - 보유 Program 목록 (활성/비활성)                       │
│     - 전체 미입력 현황 요약                                 │
│     - [+ 새 사업 만들기] 버튼                               │
└──────────────────────┬──────────────────────────────────────┘
                       │ [+ 새 사업 만들기]
                       ▼
┌─────────────────────────────────────────────────────────────┐
│ [A-3] Program 기본 정보 설정                                │
│     ◆ 사업 기본값을 정하는 단계                             │
│                                                             │
│  - 사업명 (예: "2026 중장기발전계획")                       │
│  - 사업 ID (자동 생성 또는 수동 입력)                       │
│  - 설명                                                     │
│  - 생성 부서 (자동 기록)                                    │
│  - 운영 주관 부서 (ownerOrgId)                              │
│  - 상태 (활성/비활성)                                       │
│  - 저장 시 운영 주관 부서에 기본 `program_admin` 부여       │
│                                                             │
│  [저장 후 다음 단계로] ──────────────────────────────────→  │
└─────────────────────────────────────────────────────────────┘
                       │
                       ▼
┌─────────────────────────────────────────────────────────────┐
│ [A-3-1] 연도별 운영본 생성 (ProgramVersion)                 │
│     ◆ 실제로 해당 연도에 운영할 과제/지표 세트를 고정하는 단계│
│                                                             │
│  생성 방식 선택:                                            │
│  ┌────────────────┬────────────────┬──────────────────┐    │
│  │ [빈 운영본]    │ [템플릿에서 생성] │ [전년도 불러오기] │    │
│  └────────────────┴────────────────┴──────────────────┘    │
│                                                             │
│  - 운영본명 (예: "2026 운영본")                             │
│  - 연도/차수 표기                                           │
│  - 상태: draft → active → archived                          │
│  - 전년도 복사 시: 과제 추가/삭제/산식 변경은 새 운영본에서만│
│    반영되며, 전년도 운영본은 그대로 보존                    │
└─────────────────────────────────────────────────────────────┘
                       │
                       ▼
┌─────────────────────────────────────────────────────────────┐
│ [A-4] 분류 체계 구성 (ClassificationSchema)                 │
│     ◆ 지표를 어떤 구조로 묶을지 정하는 단계                 │
│     ⚠️ 지표 생성 전에 반드시 먼저 설정해야 함               │
│                                                             │
│  Step 1. 레벨 수 선택                                       │
│  ┌──────────────────────────────────────────────┐          │
│  │  ○ 레벨 없음 (전체 단일 목록)               │          │
│  │  ● 레벨 2: 영역 > 전략 > [지표]             │          │
│  │  ○ 레벨 3: 영역 > 전략 > 과제 > [지표]     │          │
│  │  ○ 레벨 4: ...                              │          │
│  └──────────────────────────────────────────────┘          │
│                                                             │
│  Step 2. 각 레벨명 입력                                     │
│  ┌─────────────┬─────────────┐                             │
│  │ 레벨 1: [영역        ]   │                             │
│  │ 레벨 2: [전략        ]   │                             │
│  └─────────────┴─────────────┘                             │
│                                                             │
│  Step 3. 분류 노드 입력 (트리 구조)                         │
│  ┌──────────────────────────────────────────────┐          │
│  │ ▼ 교육혁신 (레벨1)                           │          │
│  │   ├ 교육과정 고도화 (레벨2)  [+하위] [삭제] │          │
│  │   └ 교육성과 관리 (레벨2)   [+하위] [삭제] │          │
│  │ ▼ 연구역량 (레벨1)                           │          │
│  │   └ ...                                      │          │
│  │ [+ 노드 추가] [엑셀로 일괄 업로드]           │          │
│  └──────────────────────────────────────────────┘          │
│                                                             │
│  업로드 원칙:                                               │
│  - 분류 체계와 지표는 기본적으로 따로 업로드                │
│  - 이 단계에서는 분류 노드만 업로드                         │
│  - 각 노드는 `분류노드코드`를 가져야 하며 이후 지표 업로드  │
│    시 이 코드로 자동 매핑                                   │
└─────────────────────────────────────────────────────────────┘
                       │
                       ▼
┌─────────────────────────────────────────────────────────────┐
│ [A-5] 상태 코드셋 정의 (StatusSet)                          │
│     ◆ 지표의 달성 상태를 어떻게 표현할지 정하는 단계        │
│     ◆ 관리자가 상태명/색상/아이콘을 자유롭게 설정           │
│     ⚠️ StatusSet은 Program 전체 기본값 또는 기간별로 다를 수 있음 │
│                                                             │
│  Step 1. StatusSet 생성 (기본 또는 기간 유형별)             │
│  - StatusSet명 (예: "정기점검용", "최종평가용")             │
│  - 해당 StatusSet을 사용할 기간 유형 지정 (필수)            │
│  - 또는 "모든 기간의 기본값" 설정                           │
│                                                             │
│  Step 2. 상태 정의                                          │
│  ┌────────────────────────────────────────────────────┐    │
│  │ 상태명        색상 (팔레트 12색)  아이콘           │    │
│  │ ─────────────────────────────────────────────      │    │
│  │ [달성완료  ]  [●초록  ▼]         [✓ check_circle] │    │
│  │ [진행중    ]  [●파랑  ▼]         [↗ trending_up ] │    │
│  │ [주의필요  ]  [●노랑  ▼]         [⚠ warning     ] │    │
│  │ [위험      ]  [●빨강  ▼]         [✗ cancel      ] │    │
│  │ [+ 상태 추가]                                      │    │
│  └────────────────────────────────────────────────────┘    │
│                                                             │
│  ⚠️ 색상 변경 시 영향 받는 화면 수 사전 안내               │
│  ⚠️ 이미 사용 중인 색상은 팔레트에서 회색 처리             │
│  ⚠️ 기간별로 다른 StatusSet을 사용하려면 별도로 생성       │
└─────────────────────────────────────────────────────────────┘
                       │
                       ▼
┌─────────────────────────────────────────────────────────────┐
│ [A-6] 지표 생성 (ManagedItem)                               │
│     ◆ 기준 항목과 해당 연도 운영 항목을 만드는 단계         │
│                                                             │
│  생성 방법 선택:                                            │
│  ┌───────────────────┬───────────────────┐                 │
│  │ [개별 생성]       │ [엑셀 일괄 업로드]│                 │
│  └───────────────────┴───────────────────┘                 │
│                                                             │
│  업로드 원칙:                                               │
│  - 지표 업로드 파일에는 `분류노드코드` 컬럼 포함            │
│  - 시스템이 해당 코드를 찾아 자동으로 분류 체계에 연결      │
│  - 관리자 수동으로 하나하나 분류 노드를 지정하지 않도록 설계│
│  - 분류 체계가 먼저 준비되지 않으면 지표 업로드는 실패 처리 │
│  - 업로드 또는 생성 결과는 현재 선택된 `ProgramVersion`에   │
│    `ProgramVersionItem`으로 저장                            │
│                                                             │
│  ── 개별 생성 폼 ──                                        │
│  - 소속 분류 노드 선택 (드롭다운 또는 트리)                 │
│  - 지표명                                                   │
│  - 유형: ● 정량  ○ 정성                                    │
│                                                             │
│  [정량 전용 설정]                                           │
│  - 단위 (%, 명, 건, 억원 등)                                │
│  - 측정 유형: number | ratio | boolean | range | composite  │
│  - 목표 방향: 증가 | 감소 | 특정값 근접                    │
│  - 판정 방식: higher_better | lower_better | target_range   │
│              | formula_based | manual_only                  │
│  - 수집 방법: manual | api | hybrid                        │
│  - 자동 판정 규칙 (MetricFormula) 설정                      │
│                                                             │
│  [정성 전용 설정]                                           │
│  - 입력 필드 구성 (계획 서술, 추진현황, 리스크 등)          │
│                                                             │
│  [공통 설정]                                                │
│  - 담당 부서 (ownerOrgId)                                   │
│  - 점검 주기 (MonitoringCycle): 월별 | 분기별 | 연도별 | 수시│
│  - 사용 StatusSet 연결                                      │
│  - 템플릿 적용 또는 전년도 운영본 항목 복사 후 수정 가능     │
└─────────────────────────────────────────────────────────────┘
                       │
                       ▼
┌─────────────────────────────────────────────────────────────┐
│ [A-6-1] 사용자 입력창 구성                                  │
│     ◆ 사용자가 보게 될 입력 화면을 관리자가 자유롭게 설계   │
│                                                             │
│  구성 요소:                                                 │
│  - 입력 필드(field)                                         │
│  - 안내 카드(info_card)                                     │
│  - 경고 카드(warning_card)                                  │
│  - 일반 설명문(plain_text)                                  │
│  - 구획 제목(section_title), 구분선(divider)                │
│                                                             │
│  편집 방식:                                                 │
│  - 블록 추가                                                │
│  - 드래그 앤 드롭 정렬                                      │
│  - 좌우/상하 위치 조정                                      │
│  - 특정 지표/과제 또는 분류 노드에만 조건부 적용            │
│                                                             │
│  예시:                                                      │
│  - warning_card: "최종 확정 후 수정이 제한됩니다."          │
│  - plain_text: "실적은 학기 종료일 기준으로 작성합니다."    │
│  - info_card: "API 값이 없으면 수기 입력 후 사유를 기록"    │
└─────────────────────────────────────────────────────────────┘
                       │
                       ▼
┌─────────────────────────────────────────────────────────────┐
│ [A-7] 기간 생성 (Period)                                    │
│     ◆ 언제 계획/실적을 입력받을지 기간을 정하는 단계        │
│     ⚠️ 기간은 단순 날짜 범위가 아니라 typeCode 기반         │
│        → 부서/항목/주기별로 실제 입력 가능한 기간이 다름   │
│     ⚠️ 기간은 현재 선택된 ProgramVersion 기준으로 생성      │
│                                                             │
│  Step 1. 기간 유형 선택 (필수)                              │
│  - typeCode: 연도 | 학기 | 분기 | 특정 차수 | 수시          │
│  - ⚠️ 기간 유형에 따라 적용할 StatusSet이 달라질 수 있음   │
│                                                             │
│  Step 2. 기간 기본 정보                                     │
│  - 기간명 (예: "2026-1차 점검기간", "2026-1학기")           │
│  - 시작일 / 마감일                                          │
│  - 기간 설명                                                │
│                                                             │
│  Step 3. 범위 및 주기 매핑                                  │
│  - 이 기간에 입력 가능한 항목 범위 지정                     │
│    (전체 항목 or 특정 분류 노드 범위)                      │
│  - 각 항목의 점검 주기(MonitoringCycle)와의 교집합 계산     │
│  - 예: 항목 주기가 "분기별"인데 기간이 "월별"이면          │
│    → 3개월 단위로만 입력 가능                              │
│                                                             │
│  ┌───────────────────────────────────────────────┐         │
│  │ 기간명            시작일    마감일    상태     │         │
│  │ ─────────────────────────────────────────     │         │
│  │ 2026-1차 점검기간  04/01    05/31    진행중   │         │
│  │ 2026-1학기         03/01    08/31    진행중   │         │
│  │ 2026-2차 점검기간  10/01    10/31    예정     │         │
│  │ [+ 기간 추가]                                 │         │
│  └───────────────────────────────────────────────┘         │
└─────────────────────────────────────────────────────────────┘
                       │
                       ▼
┌─────────────────────────────────────────────────────────────┐
│ [A-8] 권한 배당 (ProgramRoleBinding)                        │
│     ◆ 어느 부서가 어떤 범위를 입력/조회할지 정하는 단계     │
│     ⚠️ 기본은 부서 단위 권한, 외부 검토자는 예외적으로 개인  │
│        단위 조회 권한 허용                                  │
│                                                             │
│  권한 추가:                                                 │
│  - 대상 부서 선택 (SSO 조직 목록에서)                       │
│  - 역할: department_editor (입력 가능)                      │
│          department_viewer (조회만 가능)                    │
│          external_reviewer (제한적 조회)                    │
│  - 범위: 프로그램 전체  OR  특정 분류 노드 지정             │
│  - 해당 부서의 모든 부원이 같은 권한/범위 사용              │
│  - 외부 검토자는 user 단위로 지정하되 Program 전체 조회만 허용│
│                                                             │
│  ┌────────────────────────────────────────────────────┐    │
│  │ 부서명         역할           담당 범위             │    │
│  │ ───────────────────────────────────────────────    │    │
│  │ 교학처         editor         교육혁신 노드 전체   │    │
│  │ 연구처         editor         연구역량 노드 전체   │    │
│  │ 기획처         viewer         프로그램 전체        │    │
│  │ [+ 권한 추가]                                       │    │
│  └────────────────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────────────┘
                       │
                       ▼
┌─────────────────────────────────────────────────────────────┐
│ [A-9] (선택) API 연계 설정                                  │
│     ◆ 외부 시스템(학생정보, 재정 등)에서 자동으로 값을      │

│  - 동기화 스케줄 설정                                       │
└─────────────────────────────────────────────────────────────┘
                       │
                       ▼
┌─────────────────────────────────────────────────────────────┐

│                                                             │
│  위젯 선택 및 배치:                                         │
│  ┌────────────────┬────────────────┐                       │
│  │ 전체 달성 현황  │ 조직별 집계   │                       │
│  │ (도넛 차트)    │ (가로 막대)   │                       │
│  ├────────────────┼────────────────┤                       │

│  └────────────────┴────────────────┘                       │
│  [+ 위젯 추가]  [레이아웃 저장]                             │
└─────────────────────────────────────────────────────────────┘
                       │
                       ▼
                 ✓ 설정 완료
             → 운영 관리 모드로 전환
```

## B. 운영 관리 흐름 (상시 반복 작업)

최초 설정 완료 후, 관리자가 일상적으로 수행하는 작업들입니다.

```
[관리자 대시보드]
       │
       ├─── [B-1] 미입력 모니터링
       │         - 기간별 입력 현황 전체 조회
       │         - 부서별 미입력 건수 확인
       │         - 마감일 임박 항목 알림
       │         - 특정 부서/담당자에게 입력 독려 알림 발송
       │
       ├─── [B-2] 기간 관리
       │         - 새 기간 추가 (다음 학기, 다음 차수 등)
       │         - 기존 기간 마감일 조정
       │         - 기간 활성/비활성 전환
       │
       ├─── [B-3] 운영본 관리 / 지표 수정
       │         - 새 연도 운영본 생성
       │         - 전년도 운영본 불러오기 후 일부 과제 추가/삭제
       │         - 해당 연도 지표 정보 수정 (이름, 담당부서, 판정 기준 등)
       │         - 운영본 최종 확정 전까지만 구조/산식 변경 허용
       │         ⚠️ 이미 최종 확정된 운영본과 기입력 데이터는 그대로 보존
       │
       ├─── [B-4] 권한 관리
       │         - 담당자 변경 시 부서 권한 재배정
       │         - 운영 주관 부서 외 타 부서에 입력/조회 권한 배정
       │         - 권한 회수
       │
       ├─── [B-5] 엑셀 대량 업로드 (ImportBatch)
       │         ⚠️ 레거시 시스템 이관 아님, 주기별 대량 입력용
       │         ⚠️ 현재 선택된 ProgramVersion 기준으로 반영
       │         - 부서 담당자 일괄 계획값/실적값 업로드
       │         - 템플릿 다운로드 → 데이터 작성 → 유효성 검사 → 일괄 저장
       │         ┌──────────────────────────────────────┐
       │         │ Step 1. 템플릿 다운로드              │
       │         │ Step 2. 데이터 작성 후 파일 업로드   │
       │         │ Step 3. 유효성 검사 결과 확인        │
       │         │ Step 4. 부분 성공 시 오류 목록 검토  │
       │         │ Step 5. 확인 후 일괄 저장            │
       │         └──────────────────────────────────────┘
       │         - 업로드 이력 조회 (누가, 언제, 몇 건, 성공/실패)
       │
       ├─── [B-6] 값 확정 및 상태 최종화
       │         ★ 핵심 관리자 업무
       │         
       │         [6-1] 최종 반영값(finalValue) 결정
       │              - collectionMode가 hybrid인 항목만 해당
       │              - API 값 vs 수기 값 중 하나 선택
       │              - 선택 기준 및 사유 기록 (valueReason)
       │              - 예시:
       │                ├─ [API 채택] 학생정보 시스템 값 신뢰도 높음
       │                └─ [수기 채택] API 미연계, 담당자 입력값 확인됨
       │
       │         [6-2] 판정 최종화 (정량 항목만)
       │              - 자동판정 결과 검증
       │              - 필요 시 수기판정 적용 (v2 이후 전체 지원)
       │              - 최종판정(finalStatus) 확정
       │
       │         [6-3] ItemGroupSnapshot 관리
       │              - 지표 묶음의 기준일 snapshot 생성
       │              - 예: "2026-1차 점검" 기간 종료 시
       │                  → 해당 기간의 최종값 + 판정 결과 저장
       │              - 이후 집계/보고용 기준점으로 사용
       │
       ├─── [B-7] 활동 이력 조회
       │         - 누가 언제 어떤 값을 입력/수정했는지
       │         - 지표별 변경 이력
       │         - 값 확정/상태 변경/권한 배정 로그
       │
       └─── [B-8] 대시보드 / 보고서 조회
                 - 전체 달성 현황 (도넛 차트)
                 - 조직별 집계 (가로 막대, 드릴다운)
                 - 기간별 추이 (꺾은선)
                 - 상태 분포 (스택 막대)
                 - 미입력 현황 (테이블)
                 - 환류 대상 항목 목록
```

## 설정 단계 의존 관계

최초 설정 시 순서가 중요합니다. 각 단계는 이전 단계의 결과물에 의존합니다.

```
[A-3] Program 생성
       │
       ▼
[A-3-1] ProgramVersion 생성     ← Program 생성 후 연도별 운영본 확정
       │
       ▼
[A-4] 분류 체계 구성 (Schema)   ← Program/ProgramVersion이 있어야 생성 가능
       │
       ├─── [A-5] 상태 코드셋 (StatusSet)   ┐
       │     (기간 유형별로 다를 수 있음)  │
       │                                  ├─ 순서 주의 사항:
       │    [A-6] 지표 생성 (ManagedItem) ┘  기본 지표 구조 정의 후
       │          ← ProgramVersionItem 생성    StatusSet 다듬기 권장
       │
       ▼
[A-7] 기간 생성 (Period)        ← 지표 생성 후 입력 기간 열기
       │                           기간 유형에 맞는 StatusSet 확인
       │
       ▼
[A-8] 권한 배당                  ← 기간 생성 후 부서 담당자 접근 허용
       │
       ▼
[A-9] API 연계 (선택)           ← 지표의 collectionMode에 따라
[A-10] 대시보드 구성 (선택)     ← 언제든 설정 가능

⚠️ 설정 순서 주의:
  - StatusSet은 기간 유형(typeCode)과 연결될 수 있으므로,
    기간을 먼저 고려한 후 상태셋을 최종 정리하는 것도 가능
  - 실제 UX에서는 분류체계 → 기본 지표 → 상태셋 → 기간 순으로
    진행하는 것이 더 직관적일 수 있음
```

## 각 단계별 주요 UI 요소

### [A-3] Program 기본 정보
- 사업명, ID, 설명, 생성 부서, 운영 주관 부서, 활성 여부
- 운영 주관 부서에는 기본 `program_admin` 권한 자동 부여
- 저장 후 "다음: 분류 체계 구성" 버튼으로 안내

### [A-3-1] 연도별 운영본 생성
- 운영본명, 연도/차수, 생성 방식 선택
- 생성 방식:
  - 빈 운영본
  - 템플릿 기반 생성
  - 전년도 운영본 불러오기
- 전년도 불러오기 시 과제/지표/산식/담당부서 일부 수정 가능
- 운영본이 최종 확정되면 해당 연도 공식 구성으로 보존되며 일반 편집은 불가

### [A-4] 분류 체계 구성
- 레벨 수 선택 UI (라디오 버튼 or 슬라이더)
- 레벨명 입력 필드 (동적으로 레벨 수에 맞게 표시)
- 트리 형태 노드 편집기 (드래그 정렬, 인라인 편집)
- 엑셀 일괄 업로드 (대량 노드 생성 시)
- 업로드 파일에는 `노드코드`와 `부모노드코드` 포함
- 이후 지표 업로드의 자동 매핑 기준이 되는 `분류노드코드` 관리

### [A-5] 상태 코드셋
- 상태 목록 (추가/삭제/정렬)
- 각 상태별: 상태명 텍스트 + 색상 팔레트(12색) + 아이콘 선택
- 색상 중복 방지 (이미 선택된 색은 비활성)
- 변경 영향 범위 사전 안내

### [A-6] 지표 생성
- 분류 노드 선택 (트리 또는 드롭다운)
- 정량/정성 유형 선택에 따라 폼 동적 변환
- 점검 주기, 담당 부서, StatusSet 연결
- 실제 저장 단위는 선택된 `ProgramVersion`의 `ProgramVersionItem`
- 엑셀 일괄 업로드 (다수 지표 한 번에 생성)
- 지표 업로드 파일에는 `분류노드코드` 포함
- 업로드 시 코드 기준으로 분류 체계 자동 연결
- 분류 체계와 지표를 한 파일에서 동시에 생성하는 통합 업로드는 후속 확장 기능
- 템플릿 또는 전년도 운영본의 항목을 불러와 일부만 수정해 사용할 수 있음

### [A-6-1] 사용자 입력창 구성
- `FieldDefinition`과 별도로 안내/설명 블록을 배치할 수 있어야 함
- 블록 타입 예:
  - field
  - info_card
  - warning_card
  - plain_text
  - section_title
  - divider
- 관리자는 드래그 앤 드롭으로 필드와 안내 문구 위치를 자유롭게 조정 가능
- 특정 지표/과제, 특정 분류 노드, 특정 입력 유형(계획/실적)에만 블록을 붙일 수 있음

### [A-7] 기간 관리
- 기간 목록 테이블 (이름, 시작일, 마감일, 상태)
- 간단한 폼으로 기간 추가
- 마감일 연장 (날짜 수정)
- 기간 활성/종료 전환 버튼

### [A-8] 권한 배당
- SSO 조직 목록에서 부서 검색/선택
- 역할 드롭다운 (`program_admin` / `editor` / `viewer` / `external_reviewer`)
- 범위 선택 (전체 or 특정 노드 선택)
- 현재 배당 현황 테이블
- Program 생성 부서와 운영 주관 부서가 다를 수 있으며, 운영 주관 부서는 기본 관리자 역할을 가진다.
- 외부 검토자는 입력 권한 없이 Program 전체 제한적 조회만 허용한다.

## 관리자 GNB (Global Navigation Bar) 구성

일반 사용자(부서 담당자) GNB와 완전히 분리됩니다. (UI 원칙 P6)

```
[관리자 GNB]
  ├─ 대시보드        : 전체 현황 및 모니터링
  ├─ 사업 관리       : Program 목록, 생성, 편집
  │    └─ [사업 선택 시]
  │         ├─ 분류 체계
  │         ├─ 지표 관리
  │         ├─ 기간 관리
  │         ├─ 상태 코드셋
  │         ├─ 권한 관리
  │         ├─ API 연계
  │         └─ 대시보드 구성
  ├─ 입력 현황       : 미입력 모니터링, 독려 알림
  ├─ 업로드 이력     : ImportBatch 조회
  ├─ 활동 로그       : 전체 변경 이력 감사
  └─ 시스템 설정     : 필드 라이브러리, 공통 코드 관리
```

## 기타 중요 사항

### ItemTemplate 활용
현재 플로우에는 명시되지 않았으나, 범용 플랫폼의 핵심 기능입니다.
- **ItemTemplate**: 자주 반복되는 지표 구성을 템플릿화하여 재사용
  - 예: "부서별 진학률 조사" 템플릿 → 각 학과별로 인스턴스 생성
  - 예: "졸업생 취업률" 템플릿 → 학년별, 계열별로 인스턴스 생성
- 대량 지표 생성 시 [A-6]의 엑셀 업로드 외에도 템플릿 기반 생성 활용 가능
- 다만 템플릿만으로는 연도별 운영 차이를 보존할 수 없으므로, 실제 운영은 항상 `ProgramVersion`과 `ProgramVersionItem` 기준으로 고정한다.
- 가장 자주 쓰는 시작 방식은 `전년도 운영본 불러오기`다.

### 업로드 정책
- **기본 원칙**: 분류 체계와 지표는 분리 업로드
- **분류 체계 업로드**: 노드 구조와 코드 정의
- **지표 업로드**: 지표 정보 + `분류노드코드`로 자동 매핑
- **관리자 수동 연결 최소화**: 지표 업로드 후 개별 항목마다 분류 노드를 손으로 지정하지 않음
- **통합 업로드**: 한 파일에서 분류 체계와 지표를 동시에 생성하는 방식은 초기 복잡도가 높아 후속 확장으로 둠

### 후속 확장 (v2 이후)
현재 v1에는 포함되지 않으나, 향후 추가될 기능들:
- **제출-접수/반려 워크플로우**: 담당자 제출 → 관리자 접수 또는 반려
  - 반려 시 관리자 의견/사유 기록
- **수기판정 전체 지원**: 정량/정성 모두 관리자 수기판정 가능
- **입력 이력 조회**: 변경 전/후 값, 변경자, 변경 일시 추적
- **변경 영향 분석**: 지표 수정 시 기입력된 데이터 영향도 사전 분석
- **외부 검토자 범위 세분화**: 위원회별 조회 범위, 열람 기간, 의견 작성 기능
