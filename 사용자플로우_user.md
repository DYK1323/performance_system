# 부서 담당자 사용자 흐름 (User Flow)

## 개요

본 문서는 범용 성과관리 플랫폼에서 **일반 사용자(부서 담당자)**의 전체 이용 흐름을 정의합니다.  
사용자 제안의 "사업 중심 탐색" 구조와 기획안 v2의 "할 일 중심 진입" 구조를 통합한 최종 정제안입니다.

## 각 단계별 상세 설명

### [1] 로그인
- **요소**: 이메일/ID, 비밀번호 입력 필드, 로그인 버튼, SSO 버튼
- **선행 조건**: 사용자 계정 생성 및 부서 할당 완료
- **다음 단계**: [2] 홈 대시보드

---

### [2] 홈 대시보드 (할 일 현황)
- **목표**: 부서 담당자가 로그인 직후 "지금 무엇을 해야 하는가"를 3초 내 파악
- **핵심 요소**:
  - 현재 활성 기간 배너 (예: "2026년 1학기 실적 입력 기간 | 마감 D-7") — 시각적 강조
  - 미입력 항목 카운트 카드 (계획/실적 분리)
  - 담당 사업별 입력 완료율 (Progress Bar)
  - 최근 활동 기록
  - 공지사항/알림 영역
- **진입 경로**: 
  - [담당 사업 선택] → [3] 담당 사업 목록
  - [미입력 항목 바로가기] → [4-1] 미입력 항목 목록
- **UI 원칙 참고**: P3 (담당 항목 우선 노출), P2 (즉각 피드백)

---

### [3] 담당 사업 목록
- **목표**: 자신이 담당하는 사업과 현재 운영본을 선택하고 현황을 한눈에 파악
- **핵심 요소**:
  - 배당된 Program 카드 또는 리스트
  - 사업별 기간별 입력 완료율 (계획/실적 분리)
  - 기간 필터 드롭다운 (관리자가 생성한 Period 목록) — 전역 컨텍스트
  - 사업명, 운영 중 버전/연도, 담당자 정보
- **액션**: 사업 카드 클릭 → [4] 사업 내 지표 목록

---

### [4] 사업 내 지표 목록
- **목표**: 어떤 지표에 무엇을 입력해야 하는지 파악 및 선택
- **핵심 요소**:
  - Breadcrumb 네비게이션 (대시보드 > 사업명 > 운영본 > 분류체계)
  - 현재 운영본과 현재 기간 항상 상단 고정 표시
  - 계획 입력 현황 컬럼 / 실적 입력 현황 컬럼 (좌우 분리)
  - ClassificationNode 계층 탐색 (아코디언 또는 트리)
  - ProgramVersionItem 목록
  - 각 항목의 입력 상태 배지:
    - 미입력 (○) — 회색
    - 임시저장 (◐) — 노랑
    - 완료 (✓) — 초록
    - 확정대기 (◐) — 파랑
- **액션**: 계획/실적 셀 클릭 → [5] 입력 화면

---

### [5] 입력 화면
- **목표**: 실제 값과 근거를 입력하고 저장
- **상단 컨텍스트 영역**:
  - Breadcrumb (사업명 > 운영본 > 분류체계 > 지표명)
  - 현재 운영본, 현재 기간, 입력 유형(계획/실적) 표시
- **입력 폼 주요 필드**:
  - 지표명/단위 (읽기 전용)
  - 목표값 (참고용 표시, 계획 입력 시에는 목표값 자체를 입력)

  **[정량 실적 입력 시 - API/Manual/Hybrid 구조 반영]**
  - API 연계 여부에 따라 입력 필드 구성이 다름:
    - **manual** (수기 입력만): 입력값 1개 필드
    - **api** (자동 조회): API 값 (읽기 전용) 표시, 수기 입력 불가
    - **hybrid** (API 우선, 수기 대체 가능):
      - API 값 표시 (자동 조회된 값, 읽기 전용)
      - 수기 값 필드 (API 실패/미연계 시에만 입력, 선택사항)
      - 수기 대체 시 사유 기록 (필드: valueReason)
      - finalValue는 관리자 또는 운영 규칙에 따라 최종 확정

  - ★ 비고/설명 (선택사항)
  - ★ 증빙 파일 첨부 — 첨부 여부는 항목별 정책에 따라 필수/선택
    - 다중 파일 지원, 드래그앤드롭
    - 현재본만 관리 (이력 조회 기능 없음)

- **액션 버튼**:
  - 임시저장 (입력 중 자동 저장 가능 검토)
  - 최종저장 (저장 완료 후 [6]으로 이동)
  - 취소 (목록으로 돌아가기, 임시저장 데이터 유지)
- **UI 원칙 참고**: P1 (이중 입력 경로), P2 (즉각 피드백)

---

### [6] 저장 완료 + 상태 피드백
- **목표**: 저장 완료와 판정 결과를 명확하게 전달하여 사용자 불안감 해소
- **현재 운영 모드**: 입력 후 즉시 반영 (단순 제출-접수/반려는 후속 확장)
- **핵심 요소**:
  - 저장 완료 메시지 (토스트 또는 인라인)
  - **정량 항목의 자동판정 결과 표시** (정성 항목은 상태 판정 없음):
    - finalValue 기반 판정
    - hybrid 항목에서 관리자 확정 전이면 "최종값 확정 대기" 안내 가능
    - 결과: 달성 (초록) / 미달성 (빨강) / 판정불가 (회색) / 판정기준 없음
    - 판정 근거 간단히 표시 (예: "85% ≥ 목표 85%")
  - [판정 기준 보기] 링크 (상세 설명)
  - **상태 최종 확정은 현재 자동 판정만 사용** (수기 판정은 관리자만 가능, 후속 확장)
- **액션 버튼**:
  - [다음 미입력 항목으로] — 연속 입력 UX 지원
  - [목록으로 돌아가기] — [4]로 복귀
- **UI 원칙 참고**: P2 (즉각 피드백), P4 (판정 투명성)

## 핵심 설계 결정 사항

| 설계 결정 | 내용 | 근거 |
|---------|------|------|
| **운영본과 기간은 전역 컨텍스트** | 별도 선택 화면 없이 항상 상단 표시, 필터로 변경 | 사용자는 같은 Program 안에서도 현재 어느 연도 운영본을 보고 있는지 알아야 함 |
| **계획/실적 구분은 컬럼** | 지표 목록에서 좌우 컬럼 분리, 직접 클릭 진입 | 클릭 depth 최소화, 입력 속도 향상 |
| **대시보드 진입점 우선** | 로그인 직후 할 일 현황을 먼저 노출 | 누락 방지 및 사용자 중심 설계 |
| **증빙은 입력 화면에 통합** | 별도 단계 없이 [5]에서 함께 처리 | 대학 행정 특성상 필수, 입력-증빙 동시 처리 |
| **저장 후 자동판정 피드백** | [6]에서 판정 결과 즉시 표시 | 사용자가 입력 완료 여부를 명확히 인식 |

## 기타 사항

### 대시보드에서 조회 가능한 정보
사용자는 입력 흐름 외에도 다음을 대시보드에서 조회할 수 있습니다:
- 담당 사업별 입력률
- 시간대별 달성률 추이
- 내 권한 범위 상태 분포
- 미입력 알람

### 모바일 고려사항
- 모바일은 **조회 중심** (대시보드, 목록 조회)
- 단건 입력은 제한 지원 (수정, 빠른 추가만)
- 엑셀 업로드, 스키마 설정은 데스크톱 전용

### 접근성
- UI 원칙 P5, P8 참고
- 모든 상태는 색상 외 아이콘+텍스트로 표현
- 키보드 네비게이션 지원

### 후속 확장 기능 (v2 이후)
현재 v1에는 포함되지 않으나, 향후 추가될 기능들:
- **이력 조회**: 입력값 변경 이력, 판정 변경 이력 (감시/감사)
- **제출-접수/반려 워크플로우**: 부서 담당자 제출 → 관리자 접수 또는 반려
  - 반려 시 관리자 의견 확인 후 수정 가능
- **수기판정 요청**: 담당자가 자동판정 결과에 이의 제기 (관리자 최종 판단)

## 주요 설계 원칙 재확인

### 현재(v1) vs 후속(v2+) 운영 모드 차이

| 항목 | v1 (현재) | v2+ (후속) |

### API/Manual/Hybrid 입력 구조의 의미

정량 실적 입력 시 데이터 원천에 따라 사용자 UX가 달라집니다:
- **manual**: 사용자가 직접 입력
- **api**: 시스템이 자동 조회, 사용자는 확인만
- **hybrid**: API 우선 조회, 실패 시 사용자가 수기 입력 + 사유 기록
