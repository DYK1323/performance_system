# 범용 성과관리 플랫폼 화면 목록표

## 개요

이 문서는 [사용자플로우_admin.md](c:/Users/DAYOUNG/performance_system/사용자플로우_admin.md)와 [사용자플로우_user.md](c:/Users/DAYOUNG/performance_system/사용자플로우_user.md)를 기준으로 도출한 화면 목록표이다.

- 목적: 화면 단위를 확정하고, 각 화면의 역할·주요 기능·진입 경로를 한눈에 정리
- 범위: v1 기준 실제 구현 대상 화면 중심
- 원칙:
  - 관리자 화면과 부서 담당자 화면을 분리
  - 팝업/모달이 핵심 기능을 담당하면 별도 화면 단위로 기록
  - 후속 확장 기능은 별도 구분

## 화면 분류 체계

- `ADM`: 관리자 화면
- `USR`: 부서 담당자 화면
- `COMMON`: 공통 화면
- `MODAL`: 별도 설계가 필요한 모달/팝업성 화면

## v1 설계 원칙 및 메모

### 사용자 화면 (v1)
- 사용자는 `조회 + 입력` 가능하되, 조회 범위는 본인 부서에 배정된 권한 범위로 제한
- **대시보드**: 사업 단위 행, 자신의 부서가 권한을 가진 사업만 표시
- **입력**: 정량(manual만), 정성(자유), 파일 첨부 지원
- **저장 후**: "저장됨" + 판정 결과(자동판정만) + 다음/돌아가기

### 관리자 화면 (v1)
- 관리자는 `구조 설계`, `기간 설정`, `권한 부여`, `입력 모니터링`을 담당
- **대시보드**: 부서 단위 행, ownerOrgId인 자신의 사업만 기본 표시
- **운영 지표 관리**: 사업 내부에서는 운영 지표만 관리하고, 공통 지표는 시스템관리의 지표 라이브러리에서 가져온다.
- **입력폼 구성**: **기간(Period) 단위로** 다른 필드 구성 가능, 정량/정성 외 사용자 정의 입력 유형 추가 가능
- **상태 코드셋**: 사업별 설정이 아니라 시스템관리의 공통 자산으로 관리한다.
- **API 연계**: API 소스 등록은 시스템관리, 기본 필드 매핑은 지표 라이브러리에서 관리한다. (v2+)

### 데이터 및 기술
- 실제 운영 기준은 `ProgramVersion` + `Period` 조합
- 템플릿과 전년도 복사는 운영본 생성 수단
- 입력 화면은 `필드 + 안내 블록` 조합으로 구성, 드래그 앤 드롭 배치
- 증빙: 현재본만 관리 (이력은 v2+)
- 업로드: 분류 체계와 지표 업로드 분리, classificationNodeCode로 자동 매핑
