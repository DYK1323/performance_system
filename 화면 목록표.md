# 범용 성과관리 플랫폼 화면 목록표

## 개요

이 문서는 [AdminFlow.md](c:/Users/DAYOUNG/performance_system/AdminFlow.md)와 [UserFlow.md](c:/Users/DAYOUNG/performance_system/UserFlow.md)를 기준으로 도출한 화면 목록표이다.

- 목적: 화면 단위를 확정하고, 각 화면의 역할·주요 기능·진입 경로를 한눈에 정리
- 범위: v1 기준 실제 구현 대상 화면 중심
- 원칙:
  - 관리자 화면과 부서 담당자 화면을 분리
  - 팝업/모달이 핵심 기능을 담당하면 별도 화면 단위로 기록
  - 후속 확장 기능은 별도 구분

## 화면 분류 체계

- `ADM`: 관리자 화면
- `USR`: 부서 담당자 화면
- `COMMON`: 공통 화면
- `MODAL`: 별도 설계가 필요한 모달/팝업성 화면

## 화면 목록 작성 메모

- `UserFlow.md` 기준으로 사용자는 `조회 + 입력` 가능하되, 조회 범위는 본인 부서에 배정된 범위로 제한한다.
- `AdminFlow.md` 기준으로 관리자는 `구조 변경`, `기간 설정`, `권한 부여`, `최종값/상태 확정`, `집계 및 환류 조회`를 담당한다.
- 실제 운영 기준은 `ProgramVersion`이며, 템플릿과 전년도 복사는 운영본 생성 수단으로 사용한다.
- 증빙은 v1에서 `현재본만 관리`하므로 증빙 이력 전용 화면은 후속 확장으로 분리했다.
- `ItemGroupSnapshot`은 공식 판단용이므로 관리자 화면으로 별도 분리했다.
- 업로드는 `분류 체계 업로드`와 `지표 업로드`를 기본적으로 분리하고, 지표는 `classificationNodeCode`로 자동 매핑한다.
- 관리자가 지표 업로드 후 분류 체계를 하나하나 수동 지정하는 방식은 기본 운영 시나리오에서 제외한다.
